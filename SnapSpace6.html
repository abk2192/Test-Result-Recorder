Getting Started with SnapSpace

            
SnapSpace is a fully self-contained, offline-first workspace designed for capturing technical evidence, rapid note-taking, and documentation without relying on external servers.

            
            
Keyboard Shortcuts

            
Windows 10 native conventions are supported to speed up data entry:

            
    
              
  • Ctrl + V: Paste images/screenshots directly into the Notes & Media area.
    • 
              
  • Ctrl + S: Trigger the Workspace Export dialog.
    • 
              
  • Ctrl + O: Open the Import Workspace dialog.
    • 
              
  • Alt + T: Create a new tab.
    • 
              
  • Alt + N: Add a new item to the current tab.
    • 
              
  • Alt + V: Open Compare Mode side-by-side view.
    • 
            

          


          

            
Managing Items & Media

            
Items are the core building blocks of SnapSpace. They contain custom key-value fields and a rich text editor for storing visual and textual data.

            
            

              Templates Engine
              

                Clicking the Bookmark+ icon on any item header saves its current structure (Custom Fields and text/media) as a reusable Template. Access the Templates menu via the main menu to inject these saved structures into any active tab. Templates can also be exported and shared with team members as JSON files.
              

            

            

              Drag and Drop
              

                You can drag the Tab buttons at the top of the screen left and right to reorder them. You can also grab any Item by the grab indicator next to its name and drag it up or down to reorganize your view.
              

            

            

              Compare Mode
              

                Located in the main menu, Compare Mode splits the screen vertically. It allows you to select any two items from your workspace and evaluate their Notes & Media side-by-side with synchronized scrolling.
              

            

          


          

            
Import, Export & Reset

            
SnapSpace data never leaves your machine unless you explicitly export it.

            
            
Exporting

            
Clicking Export Workspace opens a checklist where you can granularly select which tabs and items to include. The output is a standalone HTML file containing a read-only viewer and a hidden JSON payload.

            
            
Importing Strategies

            
When loading a previously exported HTML file, you are given three merge strategies:

            
    
              
  • Append: Adds the imported tabs alongside your existing ones.
    • 
              
  • Merge: Drops all imported items directly into your currently active tab.
    • 
              
  • Replace: Overwrites the current board entirely with the imported data.
    • 
            

          


          

            
Architecture Overview For UI Team

            
SnapSpace is designed as a zero-dependency, single-file application. HTML, CSS (using CSS variables for theming), and vanilla JavaScript are bundled into one document. This ensures that an exported report is entirely self-sufficient.

            
            
Core Principles

            
    
              
  • No Build Step: Direct DOM manipulation using document.createElement and innerHTML.
    • 
              
  • Event Delegation: A single global click listener handles interactions based on data-* attributes.
    • 
              
  • Offline First: Everything operates via localStorage and File APIs.
    • 
            

          


          

            
State Management

            
The application relies on a single source of truth stored in memory and synced to localStorage after every mutation.

            
            
State Object Structure


let state = {
  activeTabId: "uuid-string",
  workspaceTitle: "My Project",
  tabs: [
    {
      id: "uuid-string",
      name: "Tab 1",
      scenarios: [  // Internal variable mapping to "Items"
        {
          id: "uuid-string",
          name: "Item Name",
          fields: [{ id: "uuid", key: "Env", val: "Prod" }],
          evidenceHtml: "
Rich text and images...
",
          isOpen: true
        }
      ]
    }
  ]
}


            
            
Rendering Loop

            
When the state mutates, render() is called. This triggers renderTabs() and renderPanel(), completely tearing down and rebuilding the DOM elements. Input focus states are preserved using a small timeout hack after rendering.

          


          

            
Media & File Handling

            
To prevent browser memory leaks associated with URL.createObjectURL(), files and images are never stored as temporary browser references.


            
Base64 Encoding

            
When an image is pasted or a file is attached, the FileReader API converts the binary data into a Base64 Data URL string. This string is injected directly into the HTML payload.

            
When a user deletes a file from the WYSIWYG editor using the backspace key, the DOM node is removed. The input event listener catches the DOM mutation, saves the new HTML string to the state, and the JavaScript Garbage Collector automatically clears the unreferenced memory.
